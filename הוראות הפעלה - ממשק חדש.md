# מַגִּיהַּ — הוראות הפעלה לממשק החדש

מדריך למשתמש עבור ממשק הסקירה החדש (webui): הפעלה, תצוגות, סטטוסים, סינון, ייצוא, וניהול סריקות.

---

## 1. הפעלת הממשק

### דרישות מוקדמות

- **פייתון 3.9 ומעלה** (נבדק עד 3.14). בדיקה: `python --version`
- **קובץ `report.db` מסריקה קיימת.** הממשק *מציג* ממצאים — הוא אינו יוצר אותם. אם עדיין לא הרצתם סריקה, ראו סעיף 1.1.

### הפעלה

פותחים שורת פקודה (PowerShell או cmd), עוברים **לתיקייה שבה נמצא הקוד אצלכם** ומריצים:

```
cd "C:\הנתיב\שלכם\magiah"
python -X utf8 -m magiah ui --out ..
```

> ⚠️ החליפו `C:\הנתיב\שלכם\magiah` בנתיב האמיתי במחשב שלכם — התיקייה שמכילה את `pyproject.toml` ואת תיקיית המשנה `magiah`.

- הדפדפן נפתח אוטומטית בכתובת `http://127.0.0.1:8766/`
- `--out ..` מצביע על **תיקיית הנתונים** — התיקייה שמכילה את `report.db`. הנקודותיים הכפולות (`..`) פירושן "התיקייה שמעל תיקיית הקוד". אם הנתונים נמצאים במקום אחר, כתבו נתיב מלא:
  ```
  python -X utf8 -m magiah ui --out "C:\נתיב\לתיקיית הסריקה"
  ```
- לעצירת השרת: `Ctrl+C` בחלון הפקודה.

**פורט אחר** (אם 8766 תפוס):

```
python -X utf8 -m magiah ui --out .. --port 8820
```

**בהפעלה הראשונה** נבנה מסד הנתונים המאוחד `ui_review.db` מתוך `report.db`. זה לוקח כשתי דקות ומתבצע פעם אחת בלבד. בהפעלות הבאות העלייה מיידית.

### 1.1 אין עדיין `report.db`? מריצים סריקה תחילה

אם מופיעה ההודעה **"לא נמצא קובץ report.db בתיקייה"** — זה תקין: עדיין אין סריקה בתיקייה הזו. מריצים אחת מהאפשרויות:

**סריקה על מאגר אוצריא (מסד הנתונים):**
```
python -X utf8 -m magiah all --otzaria --out ..
```

**סריקה על ריפו הספרייה (קבצי טקסט) יחד עם ספרי ספריא מהמסד — מומלץ:**
```
python -X utf8 -m magiah all --library "C:\OTZ\otzaria-library" --otzaria --out ..
```

סריקה מלאה אורכת כ־45 דקות. בסיומה מריצים שוב את פקודת ההפעלה מסעיף 1.

לחלופין, אם כבר קיימת סריקה בתיקייה אחרת — הפעילו את הממשק עם `--out` שמצביע לשם.

**הממשק הישן** ממשיך לעבוד ללא שינוי:

```
python -X utf8 -m magiah review --out ..
```
(פורט ברירת מחדל 8765). שני הממשקים מסונכרנים דרך `decisions.db`.

> הערה: השרת מאזין רק ל־`127.0.0.1` — הוא נגיש מהמחשב הזה בלבד, ללא סיסמה וללא גישה מהרשת.

## 2. מבנה המסך

- **סרגל עליון:** חיפוש חופשי, מד התקדמות, כפתור ביטול (↩), תפריט ייצוא, ניהול סריקה, סטטיסטיקה, עזרה, ומצב כהה/בהיר.
- **סרגל צד ימני:** כל המסננים.
- **אזור מרכזי:** שלוש תצוגות עבודה, נבחרות בלשוניות למעלה.

## 3. שלוש התצוגות

### 📋 תצוגת טבלה (ברירת מחדל)
טבלה מדפדפת עם כותרות בעברית. לכל כותרת יש סימן ⓘ — מעבר עכבר מציג הסבר מלא על העמודה.

- לחיצה על שורה פותחת **חלונית פרטים** בצד: הקטע לפני ואחרי התיקון, כל השדות, הערה, תיקון ידני, והיסטוריית הממצא.
- **תיבות סימון + פעולות מרוכזות:** מסמנים כמה שורות, ובסרגל התחתון בוחרים סטטוס לכולן בבת אחת. אפשר לסמן «החל על המילה בכל מקום».
- לחיצה על תגית הסטטוס בשורה פותחת תפריט מהיר לשינוי.

### 🗂 תצוגת כרטיסים
ממצא אחד במסך מלא, לסקירה מהירה עם המקלדת. מיועד למעבר על כמויות גדולות.

### 🛠 מצב מתקן
התצוגה של מי שמתקן את הספרים בפועל.

1. בוחרים ספר מהרשימה — **מוצגים רק ספרים שיש בהם ממצאים מאושרים שטרם תוקנו**, עם מספר הממצאים שנותרו בכל ספר.
2. הממצאים מסודרים **לפי סדר הופעתם בספר** (לפי מזהה השורה), ומקובצים ויזואלית לפי מראה מקום — כך עוברים על הספר מלמעלה למטה בלי לקפוץ.
3. לכל שורה: כפתורי **📋 העתק** (המילה השגויה / התיקון), וכפתור גדול **תוקן ✓**.
4. למעלה: פס התקדמות «תוקנו X מתוך Y בספר זה».

לממצאים שנסרקו מקבצי טקסט מוצג גם **נתיב הקובץ ומספר השורה**, עם כפתור להעתקת הנתיב המלא — כדי לפתוח את הקובץ הנכון ישירות.

## 4. הסטטוסים וקיצורי המקלדת

| סטטוס | סמל | משמעות | קיצור |
|---|---|---|---|
| טרם נבדק | ⬜ | מצב ההתחלה — עדיין לא הוחלט | — |
| אושר — זו שגיאה | ✅ | שגיאה אמיתית, התיקון נכון | `י` |
| תוקן בספר | 🔧 | אושר **וגם** תוקן בפועל בספר | `ק` |
| לא שגיאה | ❌ | המילה תקינה כפי שהיא | `נ` |
| דרוש בירור | ❓ | צריך בדיקה נוספת | `ב` |
| התעלם | 🚫 | לא רלוונטי, אין טעם להתלבט שוב | `ע` |

קיצורים נוספים:

| מקש | פעולה |
|---|---|
| `ד` | לא שגיאה — **בכל מקום** שבו מופיעה המילה |
| `ת` | הקלדת תיקון ידני (Enter לשמירה) |
| `רווח` | דילוג לממצא הבא, בלי החלטה |
| `Enter` | במצב מתקן: «תוקן» ומעבר לבא |
| `Ctrl+Z` | ביטול הפעולה האחרונה |
| `Esc` | סגירת חלונית או תפריט |

### מה ההבדל בין הסטטוסים?

- **אושר** מול **תוקן בספר** — «אושר» אומר "זיהינו שגיאה"; «תוקן» אומר "כבר תיקנתי אותה בקובץ". ההפרדה מאפשרת לעקוב אחרי התקדמות התיקון בפועל. שניהם נכללים בייצוא התיקונים.
- **לא שגיאה** מול **התעלם** — «לא שגיאה» היא קביעה לשונית: המילה נכונה. היא גם מלמדת את הסורק (רשימה לבנה) שלא לסמן אותה שוב. «התעלם» אומר "לא מעניין אותי", בלי לקבוע שהמילה תקינה.
- **דרוש בירור** — שומר את הממצא בצד לבדיקה עתידית. אינו נכלל בייצוא התיקונים.
- **מופע זה** מול **בכל מקום** — `נ` דוחה את המופע הבודד; `ד` יוצר כלל החל על **כל** מופעי אותה מילה במאגר כולו.

**ביטול:** כל פעולה ניתנת לביטול ב־`Ctrl+Z`. פעולה מרוכזת על 50 ממצאים מתבטלת כצעד אחד.

## 5. סינון ומיון

בסרגל הצד:

- **מאגר** — ספריא, דיקטה, אוריתא וכו'.
- **ספר** — עם חיפוש ומספר הממצאים הממתינים בכל ספר.
- **סוג שגיאה** — רשימת סימון מרובה עם מונים.
- **סטטוס** — סינון מרובה.
- **מאומתים בלבד** — רק ממצאים ברמת ודאות גבוהה.
- **ציון מינימלי** — מחוון סף.
- **סדר מיון** — ציון (גבוה→נמוך או הפוך), אקראי, לפי ספר, לפי מילה.

כל המסננים **מצטברים**. החיפוש החופשי בסרגל העליון סורק מילה, הצעת תיקון, קטע טקסט ומראה מקום.

> **סדר עבודה מומלץ:** «מאומתים בלבד» + מיון לפי ציון מהגבוה לנמוך. אלה הממצאים שכמעט תמיד מתבררים כשגיאות אמיתיות.

המצב הנוכחי נשמר בכתובת הדפדפן — אפשר לסמן במועדפים או לרענן בלי לאבד את הסינון.

## 6. ייצוא

בתפריט **⬇ ייצוא** בסרגל העליון:

### Excel — חוברת לכל מאגר
נכתב אל `<תיקיית הנתונים>\excel\שגיאות_<שם המאגר>.xlsx`. בכל חוברת:

- גיליון **סיכום** — כמויות לפי סוג שגיאה וסטטוס, והספרים עם הכי הרבה ממצאים.
- גיליון **כל השגיאות — לפי ספר** — כל ממצאי המאגר, ממוינים לפי ספר ואז לפי סוג וציון.
- **גיליון נפרד לכל סוג שגיאה**.

כל הגיליונות מוגדרים מימין לשמאל, עם שורת כותרת מודגשת ומוקפאת.

### ייצוא תיקונים ל־to_send
נכתב אל `<תיקיית הנתונים>\to_send\`:

- `approved_fixes_all.csv` + קובץ נפרד לכל מאגר — כל הממצאים בסטטוס «אושר» או «תוקן».
- `rejected_words.txt` — המילים שסומנו «לא שגיאה».

הפורמט זהה לזה של הכלי הישן (אותן 8 עמודות, קידוד UTF-8 עם BOM), כדי שהצינור הקיים ימשיך לעבוד. **תיקון ידני תמיד גובר** על ההצעה האוטומטית.

## 7. ניהול סריקה

בכפתור **⚙ ניהול סריקה**. כל פעולה מציגה חלון אישור המפרט בדיוק מה עומד לקרות.

| פעולה | מה היא עושה |
|---|---|
| 🔄 **רענן מסריקה חדשה** | טוען ממצאים מעודכנים מ־`report.db` אחרי הרצת סורק. החלטות על ממצאים שעדיין קיימים **נשמרות**; ממצאים שנעלמו מוסרים; חדשים נוספים כ«טרם נבדק». |
| 📥 **ייבוא החלטות ישנות** | מייבא את ההחלטות מ־`decisions.db` של הכלי הישן. **פעולה יזומה בלבד** — מצב ההתחלה נקי. |
| 🧹 **אפס החלטות** | מנקה סטטוסים, הערות, תיקונים והיסטוריה. **גיבוי נשמר אוטומטית** לפני האיפוס. |
| ⚠ **אפס הכל** | בנוסף מרוקן את `decisions.db`, כך שסריקות הבאות לא יושפעו מהחלטות קודמות. |

**גיבויים** נשמרים ב־`<תיקיית הנתונים>\backups\` ומופיעים ברשימה בתחתית החלון, עם כפתור **↩ שחזר** לכל אחד. השחזור מתאים החלטות לפי זהות הממצא, ולכן עובד גם אחרי סריקה מחדש.

## 8. הרצת סריקה חדשה מהממשק

בחלון ניהול הסריקה, תחת **▶ הרצת סריקה חדשה**.

### מקור הסריקה

| מצב | מה נסרק |
|---|---|
| **ריפו + ספריא מה־DB** (מומלץ) | קובצי הטקסט מהעותק המקומי של ריפו `otzaria-library` (כל המאגרים חוץ מספריא), **בתוספת** ספרי ספריא מתוך `seforim.db`. |
| **ריפו בלבד** | רק קובצי הטקסט שבריפו. |
| **DB בלבד** | כל הספרים מתוך `seforim.db`, כמו בסריקות הקודמות. |

**למה מצב משולב (hybrid)?** את הספרים שלכם אתם מעדכנים בעותק המקומי של הריפו — קובצי טקסט. ספרי ספריא, לעומת זאת, קיימים רק בתוך `seforim.db`. המצב המשולב סורק את שניהם יחד, כך שתיקונים שביצעתם בקבצים נכנסים לסריקה **מיד**, בלי להמתין לגרסה חדשה של `seforim.db`.

נתיבי ברירת המחדל:
- ריפו הספרייה: `C:\OTZ\otzaria-library`
- מסד הנתונים: `C:\ProgramData\otzaria\books\seforim.db`

### שלבים והגדרות

בוחרים אילו שלבים להריץ (מילון → כיול → איתור → מיקום → דוחות). תחת **⚙ הגדרות מתקדמות** זמין כל סף זיהוי, עם שם בעברית, הסבר וערך ברירת המחדל.

בזמן הריצה מוצג יומן חי. אפשר **⛔ לבטל** בכל רגע. בסיום מופיע כפתור **🔄 רענן ממצאים** שטוען את התוצאות החדשות לממשק.

> סריקה מלאה של המאגר האמיתי אורכת זמן רב (45 דקות ומעלה). ההגדרות נשמרות ב־`run_config.json` — אותו קובץ שמשמש גם את שורת הפקודה.

## 9. שאלות נפוצות

**הייצוא נכשל: «הקובץ נעול (כנראה פתוח באקסל)»**
הקובץ פתוח באקסל, ואקסל נועל אותו לכתיבה. סוגרים את החוברת באקסל ולוחצים ייצוא שוב. הממשק לעולם לא ידלג בשקט על קובץ נעול — הוא יציג בדיוק איזה קובץ חוסם.

**איך מוסיפים ספרים חדשים?**
מוסיפים את קובצי ה־`.txt` לעותק המקומי של ריפו `otzaria-library`, בתוך התיקייה של המאגר המתאים (שם התיקייה העליונה הוא שם המאגר). לאחר מכן מריצים סריקה במצב «ריפו + ספריא מה־DB», ובסיומה לוחצים «רענן ממצאים». ספרי ספריא מגיעים מ־`seforim.db` ואינם נוספים כקבצים.

**עדכנתי ספר — למה השגיאה עדיין מופיעה?**
הממצאים מוצגים מהסריקה האחרונה. צריך להריץ סריקה מחדש ואז «רענן ממצאים». לסימון מה שכבר תוקנתם ידנית, השתמשו בסטטוס «תוקן בספר».

**ביטלתי בטעות — אפשר לשחזר?**
`Ctrl+Z` מבטל את הפעולה האחרונה (כולל פעולה מרוכזת, כצעד אחד). לנזק רחב יותר: בניהול הסריקה יש רשימת גיבויים עם כפתור שחזור. גיבוי נשמר אוטומטית לפני כל איפוס.

**האם הממשק החדש שובר את הכלי הישן?**
לא. כל שינוי סטטוס מסונכרן אוטומטית ל־`decisions.db` בפורמט הישן, ו־`magiah review` ממשיך לעבוד. `report.db` נפתח לקריאה בלבד ואינו משתנה לעולם.

**מה ההבדל בין «ציון» ל«ציון בסיס»?**
«ציון בסיס» הוא הציון הגולמי לפי יחסי שכיחות. «ציון» הוא המשוקלל — מוסיף ראיות כמו אימות הקשר ושכיחות התיקון בספר, ומוריד ניקוד כשה«שגיאה» חוזרת בספר (סימן לכתיב מכוון). מיינו לפי «ציון».

**מה זה «מאומת»?**
ממצא שאין לו חזרות בספר, ובנוסף יש לו אימות הקשר או שהתיקון שכיח באותו ספר. אחוז הדיוק בממצאים המאומתים גבוה משמעותית — מומלץ להתחיל מהם.
