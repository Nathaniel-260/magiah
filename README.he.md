# מַגִּיהַּ (Magiah)

**איתור שגיאות הקלדה ו-OCR בטקסטים עבריים וארמיים — בלי מילונים, בלי AI.**

[English](README.md)

מגיה מוצא שגיאות בקורפוסים גדולים כמו ספריית [אוצריא](https://github.com/Sivan22/otzaria)
(7,000+ ספרים, 400+ מיליון מילים). הוא **לא משתמש במילון חיצוני ולא בבינה
מלאכותית** — וזו בדיוק הסיבה שהוא עובד גם על לשון חז"ל, ארמית, ראשי תיבות
וכתיב של אלף שנות דפוס, שאף מילון לא מכסה.

## הרעיון: הקורפוס הוא המילון של עצמו

בקורפוס של מאות מיליוני מילים, כל מילה אמיתית מופיעה פעמים רבות. שגיאה היא
כמעט תמיד **מילה נדירה מאוד שהיא שיבוש קטן של מילה נפוצה מאוד**. בנוסף,
בספרות התורנית יש כפילות פנימית עצומה — אותם ביטויים חוזרים בין ספרים,
מהדורות וציטוטים — ולכן אפשר **לאמת כל תיקון מוצע מול הקורפוס עצמו**.

### סוגי השגיאות המזוהים

| סוג | דוגמה | אימות |
|---|---|---|
| **חוסר רווח** | `אתהשמים` ← `את השמים` | הרצף המפוצל חייב להופיע *עם* רווחים במקומות אחרים בקורפוס |
| **רווח מיותר** | `הימ נו` ← `הימנו` | הצורה המאוחה נפוצה ואחד החלקים נדיר |
| **אות שגויה / חסרה / מיותרת / הפוכה** | `היעמנו` ← `הימנו` | מרחק עריכה 1 ממילה נפוצה פי 50+, עם בונוס מ**מטריצת בלבולים נלמדת** (ראו כיול), ואימות הקשר: התיקון חייב להופיע ליד אותן מילים שכנות במקום אחר |
| **אות סופית באמצע מילה** | `שלוםעליכם` | כלל דטרמיניסטי (ם ן ץ ף ך) |
| **אות רגילה בסוף מילה** | `אדמ` ← `אדם` | הצורה הסופית נפוצה פי 50+ |
| **קיצור שאיבד גרשיים** | `רמבם` ← `רמב"ם` | הצורה עם הגרשיים היא קיצור נפוץ בקורפוס |
| **שגיאות OCR ייחודיות לספר** | בלבול ד↔ר לכל אורך ספר סרוק אחד | הכיול לומד פרופיל OCR לכל ספר ומאפשר סריקה רגישה יותר בספרים עם בלבול שיטתי מוכח |
| **חילופי כתיב** (בקובץ נפרד) | `חבותיו` ↔ `חובותיו` | ו/י חסרה או מיותרת היא בדרך כלל כתיב מלא/חסר ולא שגיאה — מיוצא לקובץ משלו כדי שאתה תקבע את המדיניות |
| **סטייה מנוסח התנ"ך** (מאומת-עצמית) | פסוק שצוטט עם מילה שונה | שלשות מילים מהפסוקים נאספות מ~100 מהדורות תנ"ך; נוסח נחשב "קנוני" רק אם 2+ מהדורות מסכימות, ומהדורה בודדת שסוטה מ-3+ מהדורות מסכימות מדווחת בעצמה כחשד לשגיאה *במהדורה* — נוסח התנ"ך לעולם לא נלקח בעיניים עצומות |

כל ממצא מקבל ציון ביטחון, הדוחות ממוינים כך שהשגיאות האמיתיות מרוכזות למעלה,
ולכל סוג נוצר גם קובץ `*_verified.csv` מדויק במיוחד (מאומת-הקשר או חוזר
באותו ספר).

### מניעת התרעות שווא

מעבר לאימות הסטטיסטי, מגיה נמנע אוטומטית מהמלכודות המוכרות:

* **תחיליות מוערמות** — `דלאליעזר` (= ד+ל+אליעזר) הוא דקדוק תקין, ולכן
  "אות ראשונה מיותרת" שהיא אות שימוש (ו ה ב ל מ ש כ ד א) לעולם לא מוצעת
  כשגיאה; וכך גם סיומות נטייה ונטיות נדירות של שורשים נפוצים.
* **קיצורים** — מילים עם גרשיים (`רמב"ם`) ומילים שאחריהן גרש (`וכוננ'` =
  וכוננה) מזוהות ומדולגות.
* **קטעים בשפה זרה** — שורות שבהן יותר מדי מילים לא-נפוצות (למשל
  ערבית-יהודית בתפסיר רס"ג) מדולגות לגמרי.
* **שמות** — מילה נדירה מיד אחרי שם/רב/רבי/הרב… היא כנראה שם פרטי ומדוכאת.
* **מוסכמות של ספר** — "שגיאה" שחוזרת פעמים רבות באותו ספר היא כנראה מוסכמת
  כתיב של הספר, ומורדת בדירוג.
* **רשימה לבנה אופציונלית** (`--whitelist words.txt`) — מילים ברשימה לעולם
  לא יסומנו. דיכוי בלבד: הרשימה אף פעם לא *יוצרת* התרעות, ולכן מילון חסר
  לא יכול לפגוע בארמית או בלשון חז"ל. עובד מצוין עם רשימות הנטיות של
  [hspell](https://github.com/eyaler/hebrew_wordlists) (רישיון AGPL, ולכן
  מורידים בנפרד ולא מצורף לכאן).
* **ההחלטות שלך** — מילים שדחית בממשק הסקירה לא יסומנו שוב בסריקות הבאות,
  אוטומטית.

### שלבי העבודה

```
1. lexicon    ספירת תדירות של כל מילה                  (מעבר אחד על הקורפוס)
2. detect     סימון מילים נדירות הקרובות לנפוצות,
              אימות פיצולי רווח מול צמדי מילים          (מעבר אחד)
3. locate     איתור מופעים, זיהוי רווח מיותר,
              אימות הקשר, בדיקת ציטוטי תנ"ך            (שני מעברים)
4. report     דוחות CSV מדורגים + SQLite, פיצול לפי מאגר
5. calibrate  (אופציונלי, אחרי ריצה ראשונה) לימוד מטריצת בלבולי אותיות
              ופרופילי OCR לכל ספר — ואז detect+locate שוב לסבב חד יותר
6. review     ממשק סקירה מקומי בדפדפן
```

במחשב עם 4 ליבות, ריצה מלאה על 417 מיליון מילים (5.9 מיליון שורות, 7,293
ספרים) אורכת כ-**45 דקות**, עם ספריית Python הסטנדרטית בלבד.

## התקנה

דורש Python 3.9 ומעלה. אין תלויות חיצוניות.

```bash
pip install .
# או ישירות מהמקור:
python -m magiah --help
```

בווינדוס תמיד להריץ `python -X utf8 -m magiah …` כדי למנוע בעיות קידוד
במסוף.

## שימוש

**ספריית אוצריא** (מזהה אוטומטית את `C:\ProgramData\otzaria\books\seforim.db`):

```bash
magiah all --otzaria --out results
```

אפשר להריץ שלב-שלב (`lexicon`, `detect`, `locate`, `report`) — המקור והספים
נשמרים ב-`results/run_config.json`, כך שאחרי כוונון ספים מריצים מחדש רק
מ-`detect` בלי לבנות שוב את המילון.

**סבב שני, חד יותר** (מומלץ):

```bash
magiah calibrate --out results   # לימוד מטריצת בלבולים + פרופילי OCR
magiah detect    --out results
magiah locate    --out results
magiah report    --out results
```

**סקירת הממצאים** בדפדפן:

```bash
magiah review --out results      # נפתח בכתובת http://127.0.0.1:8765/
```

הממצאים מוצגים אחד-אחד (סינון לפי סוג שגיאה ומאגר מקור; סדר לפי ציון או
אקראי). מקלדת: **י** אישור, **נ** דחיית המופע, **ד** דחיית המילה בכל מקום,
**ת** הקלדת תיקון משלך, **ע** התעלמות לתמיד, **רווח** דילוג זמני. כל החלטה
נשמרת מיד ב-`decisions.db`; כפתור **יצוא ההחלטות** כותב תיקיית `to_send/`
עם קובץ תיקונים מאושרים לכל מאגר מקור, מוכן לשליחה למתחזקים, בתוספת
`rejected_words.txt` (שמשמש אוטומטית כרשימה לבנה בסריקות הבאות).

## הרצה על מאגר משלך

מגיה לא כבול לאוצריא. כל אוסף טקסטים בעברית/ארמית מתאים, וכל מה שצריך זה
**דרך לעבור על שורות הטקסט**. יש שלושה מתאמים מובנים, מהפשוט למתקדם.

### אפשרות 1 — תיקיית קבצי טקסט (עובד עם הכול)

שימו את הספרים בתיקייה (תת-תיקיות בסדר) של קובצי `.txt` בקידוד UTF-8, רצוי
ספר לקובץ:

```bash
magiah all --textdir path/to/books --out results
```

כל קובץ נחשב "ספר", מה שמפעיל את האימות הספר-מקומי (האם התיקון כבר בשימוש
באותו ספר?) ואת פרופילי ה-OCR.

**אם המאגר שלכם בפורמט אחר** — קובצי Word, ‏PDF עם שכבת טקסט, ‏JSON, ‏CSV,
‏HTML — הדרך הפשוטה היא להמיר לתיקיית `.txt`. למשל מ-JSON:

```python
import json, pathlib
data = json.load(open('mybooks.json', encoding='utf-8'))
out = pathlib.Path('books_txt'); out.mkdir(exist_ok=True)
for book in data:
    safe = ''.join(c if c.isalnum() else '_' for c in book['title'])
    (out / f"{safe}.txt").write_text('\n'.join(book['lines']), encoding='utf-8')
```

הטוקנייזר של מגיה כבר מסיר ניקוד, טעמים, תגי HTML וצורות תצוגה של יוניקוד —
אין צורך לנקות את הטקסט מראש.

### אפשרות 2 — כל דאטהבייס SQLite

אם המאגר כבר יושב ב-SQLite עם שורה לכל שורת טקסט:

```bash
magiah all --sqlite mycorpus.db \
    --table line --id-col id --text-col content --doc-col book_id \
    --out results
```

| דגל | משמעות | ברירת מחדל |
|---|---|---|
| `--table` | הטבלה שמכילה את שורות הטקסט | `line` |
| `--id-col` | מפתח ראשי מספרי (משמש לחלוקת העבודה בין תהליכים ולהצבעה על ממצאים) | `id` |
| `--text-col` | העמודה עם הטקסט עצמו | `content` |
| `--doc-col` | *אופציונלי אך מומלץ:* עמודה שמקבצת שורות לספרים. מפעילה אימות ספר-מקומי ופרופילי OCR | אין |

כל סכימה מתאימה כל עוד העמודות האלו קיימות. אם הטקסט מפוזר בכמה טבלאות,
אפשר ליצור VIEW:

```sql
CREATE VIEW all_lines AS
  SELECT id, book_id, text AS content FROM mishna
  UNION ALL
  SELECT id + 1000000, book_id, text FROM talmud;
```

ואז `--table all_lines`.

### אפשרות 3 — הפריסט של אוצריא

`--otzaria` הוא בסך הכול פריסט של אפשרות 2 (`table=line, id-col=id,
text-col=content, doc-col=bookId`) בתוספת העשרת דוחות שמצרפת שמות ספרים,
מראי מקום ושמות מאגרי מקור מהסכימה של אוצריא. אם `seforim.db` נמצא במקום
אחר: `--db path`.

### הערות למאגרים אחרים

* מקור הקורפוס נשמר ב-`results/run_config.json` אחרי הפקודה הראשונה —
  בשלבים הבאים אין צורך לחזור על הדגלים.
* **גודל הקורפוס משנה.** הסטטיסטיקה צריכה נפח: מתחת ל~5 מיליון מילים כדאי
  להוריד ספים (`--common-min 10 --ed1-ratio 20`) ולצפות לדיוק נמוך יותר;
  מתחת ל~מיליון מילים ההנחה "הקורפוס הוא המילון" נחלשת — שקלו להוסיף
  לקורפוס עוד טקסט מאותה סוגה (הממצאים מדווחים לפי ספר בכל מקרה, אז טקסט
  רקע נוסף לא מפריע).
* ממשק הסקירה והדוחות לפי סוג עובדים זהה בכל המתאמים. התוספות הייחודיות
  לאוצריא (תיקיות לפי מאגר מקור, מראי מקום) פשוט נשארות ריקות במאגרים
  אחרים.

### תוצרים (בתיקיית `--out`)

| קובץ | תוכן |
|---|---|
| `errors_<סוג>.csv` | קובץ מדורג נפרד לכל סוג שגיאה: מילה, תיקון מוצע, ציון, אימות הקשר, ספר, הקשר מלא |
| `errors_<סוג>_verified.csv` | תת-קבוצה מדויקת במיוחד (מאומת-הקשר או שהתיקון כבר בשימוש באותו ספר) |
| `spelling_variants.csv` | חילופי כתיב מלא/חסר — החלטת מדיניות, לא שגיאות |
| `space_errors.csv` | שגיאות רווח מיותר |
| `tanach_errors.csv` / `tanach_matches.csv` / `tanach_edition_errors.csv` | סטיות מנוסח תנ"ך מאומת-רב-מהדורות / אישורים שקטים / חשדות לשגיאות במהדורות עצמן |
| `by_source/<מאגר>/…` | אותם דוחות מפוצלים לפי מאגר המקור (בקורפוס אוצריא) |
| `report.db` | הכול כ-SQLite לשאילתות |
| `to_send/` | נכתב מממשק הסקירה: תיקונים מאושרים לכל מאגר מקור, מוכנים לשליחה |

### כוונון

כל הספים הם דגלי CLI (ראו `magiah --help`). העיקריים:

| דגל | ברירת מחדל | משמעות |
|---|---|---|
| `--rare-max` | 2 | מילה "חשודה" = עד N מופעים בקורפוס. הגדלה ל-3–5 תמצא יותר שגיאות עם יותר התרעות שווא |
| `--common-min` | 30 | תדירות מינימלית לתיקון מוצע |
| `--ed1-ratio` | 50 | פי כמה התיקון חייב להיות נפוץ מהשגיאה |
| `--workers` | 3 | תהליכים מקבילים |

## למה לא מילון? למה לא AI?

* **מילונים** נכשלים בלשון חז"ל, ארמית, מילים שאולות מיידיש, ראשי תיבות
  ואלף מוסכמות כתיב של אלף שנות דפוס. התפלגות התדירויות של הקורפוס עצמו
  *היא* המילון הנכון עבורו.
* **מודלי שפה** על 400 מיליון מילים הם איטיים, יקרים, וממציאים תיקונים.
  ראיות סטטיסטיות (יחסי תדירות + אימות צמדים והקשר) הן שחזוריות, מוסברות,
  ורצות על מחשב נייד.
